=== Q Describe you solution.
===   (free form, up to 500 words, don't need to use all 500!)
I implemented user‐level exec() by borrowing the overall structure of spawn() but staging the new program entirely in a reserved temporary region (starting at ETEMP, defined as 0xe0000000) and then handing control to a single syscall that atomically swaps in the new image.
 First, I open() the target binary and read its ELF header into a small stack buffer to verify e_magic and locate the program‐header table. Then I set a sliding pointer tmp = ETEMP and loop over each Phdr with p_type == ELF_PROG_LOAD, computing page‐aligned addresses with PGOFF() and ROUNDUP(), assigning PTE_P|PTE_U plus PTE_W if the segment is writable, and calling my map_segment(0, PGOFF(ph->p_va) + tmp, ph->p_memsz, fd, ph->p_filesz, ph->p_offset, perm) helper to either zero‐allocate new pages or read in initialized data at tmp.
 After loading all segments contiguously in the temporary region, I invoke init_stack(0, argv, &tf_esp), which uses UTEMP to build a stack page containing the argument strings, argv pointers, and argc, then records the child‐valid stack pointer. 
Finally, I call sys_exec(elf->e_entry, tf_esp, (void*)(elf_buf + elf->e_phoff), elf->e_phnum) so the kernel can tear down the old address space and map in the new one directly from ETEMP plus the provided program‐header array. 
The execl() wrapper mirrors spawnl() by counting its NULL‐terminated varargs into a const char* argv[] array and then calling exec().
 This approach avoids in‐place overwrites, leverages the kernel for the heavy lifting, and keeps the user‐level code simple and robust.
